1. Introduction
ShapeTool is a SOAP-based web service developed following the contract-first approach. The service calculates the area of five geometric shapes: Circle, Square, Rectangle, Parallelogram, and Triangle.
This project demonstrates key SOA concepts including:

XML Schema (XSD) design with validation
WSDL contract definition
Document/Literal messaging style
SOAP request/response handling
Input validation and SOAP Fault management

The service was implemented using Spring Boot + Spring Web Services (Spring-WS) in Java.

2. Design Decisions

Contract-First Approach: The XSD schema was designed first to define the exact structure of all requests and responses.
Separate Operations: Each shape has its own dedicated operation (as required by the assignment).
Document/Literal Style: Used for better interoperability and compliance with modern SOAP standards.
Namespace: http://www.woldia.edu/shapetool
Validation: Both schema-level (XSD restrictions) and application-level validation.


3. XSD Design and Schema Structure
The XML Schema (shapetool.xsd) is located at:
src/main/resources/xsd/shapetool.xsd
Key Features of the XSD:

Target Namespace: http://www.woldia.edu/shapetool
Custom Type: PositiveDouble – restricts values to numbers greater than 0 (prevents negative and zero inputs)
Complex Types: Separate request and response elements for each operation
ElementFormDefault: qualified



4. WSDL Description
The WSDL file is automatically generated by Spring-WS from the XSD at runtime.
WSDL Location (when running):
http://localhost:8080/ws/shapetool.wsdl
Main Components:

PortType: ShapeToolPortType
Operations: 5 operations (circleArea, squareArea, rectangleArea, parallelogramArea, triangleArea)
Binding: SOAP 1.1 with document/literal style
Service Endpoint: /ws/shapetool

All messages use document/literal wrapped style for clean SOAP envelopes.

5. Service Operations (All Endpoints)
The web service exposes the following five independent operations:

circleArea
Input: radius (positive double)
Formula: π × r²
Output: area and shape name

squareArea
Input: side (positive double)
Formula: side²
Output: area and shape name

rectangleArea
Input: length, width (positive doubles)
Formula: length × width
Output: area and shape name

parallelogramArea
Input: base, height (positive doubles)
Formula: base × height
Output: area and shape name

triangleArea
Input: base, height (positive doubles)
Formula: ½ × base × height
Output: area and shape name


Endpoint URL:
http://localhost:8080/ws/shapetool

Endpoint Class: ShapeToolEndpoint.java – contains all 5 @PayloadRoot methods
Service Layer: ShapeService.java – contains business logic and validation
Packages:
com.shap.area.config
com.shap.area.endpoint
com.shap.area.service
com.shap.area.model (JAXB generated classes)



7. Validation Strategy
Two-Level Validation:

Schema Validation (XSD)
PositiveDouble type rejects ≤ 0 values automatically
Missing elements or wrong types are rejected by JAXB/Spring-WS

Application-Level Validation
Extra safety check in ShapeService.java
Throws IllegalArgumentException → converted to SOAP Fault



8. SOAP Fault Handling
When invalid input is received (negative number, zero, or missing parameter), the service returns a proper SOAP Fault message.
Example Fault Scenario: Sending negative radius
Fault Code: Client
Fault String: “Radius must be greater than zero”
This demonstrates robust error handling as required by the assignment.
